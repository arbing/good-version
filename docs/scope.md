# 好版本首版功能边界

## 首版必须包含

### 项目列表

- 添加项目文件夹
- 展示多个项目
- 进入单个项目
- 修改项目显示名
- 打开项目文件夹

### 初始保护

- 项目加入后自动保存「初始好版本」
- 已有 `.git/` 时复用当前分支
- 无 `.git/` 时初始化本地 Git 仓库

### 保存好版本

- 主按钮：「保存当前好版本」
- 手动触发保存
- 默认按时间生成版本标题
- 支持选填一句说明
- 保存所有未被内部排除规则排除的文件
- 每个好版本创建 commit
- 每个好版本创建内部 tag

### 时间线

- 时间线卡片展示所有好版本
- 展示当前版本标记
- 展示保存时间和说明
- 展示新增 / 修改 / 删除文件数量
- 展示可视化文件列表
- 不隐藏 reset 后的历史版本

### 回退

- 卡片按钮：「回到这里」
- 回退前二次确认
- 回退前自动保存「回退前状态」
- 保存成功后执行 hard reset
- 回退前状态保留在时间线里
- 可再次回到回退前状态

### 导出

- 导出当前项目副本
- 包含当前项目全部文件
- 包含 `.env`
- 不包含 `.git/`
- 不包含版本历史

### 存储

- 展示版本占用大小
- 不提供清理能力

## 首版明确不做

### 协作与云

- 云同步
- 团队共享
- 账号系统
- 权限管理
- GitHub 登录
- 推送远端
- PR / Review 流程

### Git 高级能力

- 分支管理
- 合并
- 冲突解决
- rebase
- cherry-pick
- stash
- Git 教学
- 展示 commit / branch / tag 等术语

### 自动化

- 文件变化自动保存
- 定时保存
- Agent 每步自动保存
- 自动识别项目启动命令
- 自动运行测试
- 自动判断项目是否可用

### 版本管理增强

- 完整代码 diff
- 版本搜索
- 删除项目
- 删除旧版本
- 清理存储
- Git LFS
- 单文件恢复
- 局部恢复

### 其他

- 多语言
- 用编辑器打开项目
- 插件形态
- Web 应用形态
