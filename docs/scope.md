# 好版本首版功能边界

本文以当前代码实现为准，描述第一版已经开放给用户的能力和仍不包含的范围。

## 第一版已包含

### 项目列表

- 添加项目文件夹
- 拖入一个或多个项目文件夹
- 重复项目自动跳过并切换到已添加项目
- 展示多个项目
- 进入单个项目
- 修改项目显示名
- 打开项目文件夹

### 初始保护

- 项目加入后自动保存「初始好版本」
- 已有 `.git/` 时复用当前分支
- 无 `.git/` 时创建外置 Git 数据目录
- 好版本创建的新仓库，项目目录误删后仍保留外置保存数据

### 保存好版本

- 主按钮：「保存当前好版本」
- 手动触发保存
- 默认按时间生成版本标题
- 支持选填一句说明
- 保存所有未被内部排除规则排除的文件
- 每个好版本创建 commit
- 每个好版本创建内部 tag
- 当前无变化时不创建新的普通好版本

### 时间线

- 时间线卡片展示所有好版本
- 展示当前版本标记
- 展示保存时间和说明
- 展示新增 / 修改 / 删除文件数量
- 查看变化抽屉展示文件列表
- 不隐藏 reset 后的历史版本

### 回退

- 卡片按钮：「回到这里」
- 回退前二次确认
- 当前有未保存变化时，回退前自动保存「回退前状态」
- 保存成功后执行 hard reset
- 回退前状态保留在时间线里
- 可再次回到回退前状态
- 当前无变化时直接回退，不制造空版本

### 导出

- 每个好版本可导出为 `.zip` 版本压缩包
- 压缩包包含对应好版本保存时的项目文件快照
- 压缩包内包一层项目目录
- 包含 `.env`、隐藏文件和普通项目文件
- 不包含 `.git/` 目录或 `.git` 指针文件
- 不包含版本历史
- 同名压缩包已存在时自动生成不重名文件
- 导出成功后打开导出目录
- 当前项目副本导出能力已在后端保留，当前用户界面不作为正式入口暴露

### 存储

- 展示项目占用大小
- 展示版本数据占用大小，包括外置 Git 数据目录占用
- 不提供清理能力

### 项目目录丢失

- 项目路径不存在时展示丢失状态
- 支持重新选择项目目录
- 外置保存数据项目选择空目录后，可恢复最近的好版本并更新路径
- 已有 `.git/` 项目不承诺额外恢复

### 本地数据目录

- 应用展示“所有数据都保存在本地设备中”入口
- 用户可从该入口打开本地数据目录
