# 好版本 PRD

## 1. 背景

越来越多非技术用户开始使用 Coding Agent 创建网页、小工具和原型。Agent 能快速生成大量代码，但用户往往不会 Git，也不知道如何安全保存当前可用状态。

常见问题是：上一步项目还能运行，下一步 Agent 修改后项目坏了。用户不知道如何回到之前的状态，只能求助 Agent 或重新开始。

「好版本」希望解决这个问题：让不会 Git 的用户也能保存和回到项目的可用状态。

## 2. 产品目标

让非技术用户在使用 Coding Agent 修改项目时，能用简单、安全、低压力的方式完成本地版本管理。

核心目标：

- 保存当前能用的项目状态
- 查看每个版本大概改了什么
- 项目坏了时一键回到旧版本
- 回退前自动保护当前状态
- 全程不需要理解 Git

## 3. 目标用户

### 主用户

非技术个人用户，包括产品经理、运营、设计师、独立创作者、创业者。

### 次用户

小团队里的非技术成员，负责用 Coding Agent 快速搭建和修改项目。

## 4. 用户故事

### 添加项目

作为一个不会 Git 的用户，我希望选择一个项目文件夹后，工具自动开始保护它，这样我不用学习任何配置。

### 保存好版本

作为一个正在用 Agent 改项目的用户，我希望项目能跑的时候点一下「保存当前好版本」，这样后面改坏了还能回来。

### 查看变化

作为一个非技术用户，我希望看到这次版本新增、修改、删除了哪些文件，这样我能大概知道变化范围。

### 回到旧版本

作为一个项目被改坏的用户，我希望点「回到这里」就能回到之前的好版本，并且不用担心当前状态彻底丢失。

### 导出项目

作为一个要发给别人或备份项目的用户，我希望导出当前项目副本，但不带复杂版本历史。

## 5. 功能需求

### 5.1 项目列表

- 展示已添加项目
- 添加项目文件夹
- 进入项目详情
- 修改项目显示名
- 打开项目文件夹

### 5.2 自动创建初始好版本

用户添加项目后，工具自动保存一个「初始好版本」。

### 5.3 保存当前好版本

用户点击主按钮后保存当前项目状态。

保存时支持：

- 自动生成时间标题
- 可选一句说明
- 不填说明也能保存

### 5.4 时间线

项目详情页展示时间线卡片。

卡片包含：

- 保存时间
- 说明
- 当前版本状态
- 变化摘要
- 文件列表
- 查看变化
- 回到这里

### 5.5 查看变化

展示新增、修改、删除文件数量和文件列表。

不展示代码 diff。

### 5.6 回到这里

用户点击后出现二次确认。

确认后：

1. 自动保存当前状态为「回退前状态」
2. 回到目标好版本
3. 时间线继续保留回退前状态

### 5.7 导出当前项目副本

用户可以导出当前项目副本。

导出结果：

- 包含当前项目全部文件
- 包含 `.env`
- 不包含 `.git/`
- 不包含版本历史

### 5.8 存储占用

展示版本占用大小，不提供清理功能。

## 6. 非功能需求

### 易用性

- 所有核心文案使用非技术语言
- 不出现 Git、commit、branch、tag 等概念
- 主路径 3 分钟内完成

### 可靠性

- 回退前必须保存当前状态
- 保存失败不能执行回退
- 每个好版本都必须可找回

### 本地优先

- 不做云同步
- 不自动上传
- 不自动推送远端

### 跨平台

- 支持 macOS
- 支持 Windows

## 7. 风险

### 敏感文件风险

产品决策是静默保存 `.env`，且不提示风险。实现必须保证工具自身绝不上传、不推送。

### 大文件膨胀风险

MVP 全部保存到普通 Git，可能导致项目体积增长。首版只展示占用大小。

### 既有 Git 项目风险

如果项目已有 `.git/`，工具会复用当前分支并创建 commit，可能让本地分支 ahead。首版接受这个结果，但不展示 Git 术语。

## 8. 验收标准

一名不会 Git 的用户可以在 3 分钟内完成：

1. 添加项目
2. 看到初始好版本
3. 保存当前好版本
4. 查看变化摘要
5. 回到旧版本
6. 再回到回退前状态

完成过程中不需要使用命令行，不需要知道 Git 概念。
