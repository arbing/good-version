# 好版本 MVP 需求说明

## 产品定位

「好版本」是一个面向非技术用户的本地桌面版本管理工具。它帮助用户在使用 Coding Agent 创建和修改项目时，手动保存“当前能正常工作的好版本”，并在项目被改坏或项目文件夹被移动、删除后，安全回到之前的好版本。

界面不暴露 Git 概念，底层使用 Git 实现版本保存和回退。

## 目标用户

### 主目标用户

非技术个人用户：产品经理、运营、设计师、独立创作者，以及使用 Coding Agent 做网页、小工具、原型的人。

### 次目标用户

小团队里的非技术成员：负责用 Agent 快速搭建原型、不熟悉 Git、需要本地保护项目进度的人。

## 核心场景

用户用 Coding Agent 创建一个新项目。某一步项目可以正常运行，用户点击「保存当前好版本」。后续 Agent 修改项目后，项目坏了、跑不起来、界面变乱，用户打开时间线，选择之前的好版本，点击「回到这里」。

工具会先保存当前未保存状态，再把整个项目回退到目标好版本。回退前状态也会出现在时间线里，用户可以再回到回退前。

## MVP 成功标准

非技术用户能在 3 分钟内完成完整闭环：

1. 添加一个新项目
2. 自动生成初始好版本
3. 修改项目后保存当前好版本
4. 查看变化摘要
5. 回到旧好版本
6. 再回到回退前状态
7. 导出某个好版本压缩包

## 平台与形态

- 桌面应用
- 支持 macOS + Windows
- Tauri 桌面应用 + 内置 Git 操作层
- 应用内置 Git 能力，不要求用户安装 Git
- 首版只做中文界面
- 视觉风格：安心清晰、低压力

## 项目管理

- 支持多个项目列表，每次进入一个项目管理
- 用户选择项目文件夹后加入项目列表
- 用户可拖入一个或多个项目文件夹
- 重复添加的项目会被跳过，并切换到已添加项目
- 加入后自动保存一个「初始好版本」
- 支持修改项目显示名，不修改真实文件夹名
- 支持打开项目文件夹
- 首版不做删除项目功能

如果项目没有 `.git/`：

- 由好版本创建外置 Git 数据目录
- 外置 Git 数据目录保存在应用数据目录中
- 项目目录只作为当前工作区
- 项目目录丢失后，仍可从好版本恢复到重新选择的空目录

如果项目已有 `.git/`：

- 复用已有 `.git/`
- 只操作当前分支
- 不切换分支
- 不合并
- 不推送
- 不修改远端
- 项目目录丢失时不承诺额外恢复

## 保存好版本

主按钮文案：

> 保存当前好版本

保存规则：

- 手动保存
- 默认自动生成时间
- 用户可以选填一句说明
- 不填也能保存
- 保存所有未被工具内部排除规则排除的文件
- 当前无变化时，不创建新的普通好版本
- 大文件全部保存进普通 Git
- `.env` 等敏感文件静默保存
- 不弹风险提示
- 不自动推送远端

每次保存底层行为：

1. 收集当前项目状态
2. 将所有应保存文件加入 Git 索引
3. 创建一个 Git commit
4. 为该好版本创建内部 tag
5. 在应用元数据中记录版本信息

界面只显示「好版本」，不显示 commit、tag、branch 等 Git 概念。

## 时间线

版本列表使用时间线卡片。每张卡片展示：

- 保存时间
- 用户说明，没有则显示默认标题
- 是否为当前版本
- 新增 / 修改 / 删除文件数量
- 查看变化
- 导出压缩包
- 回到这里

首版不做版本搜索。时间线保留所有好版本记录，包括 hard reset 之后不在当前 Git 线性历史上的版本。

## 查看变化

首版只做轻量变化查看：

- 新增文件数量
- 修改文件数量
- 删除文件数量
- 文件列表
- 文件状态图标或颜色

首版不展示完整代码 diff。

## 回到这里

用户点击「回到这里」时必须二次确认。

建议文案：

> 会先保存当前状态，然后回到这个好版本。你之后也可以再回到现在。

回退流程：

1. 当前有未保存变化时，保存当前状态为「回退前状态」
2. 为该状态创建 commit 和内部 tag
3. 确认目标好版本的内部 tag 存在
4. 执行 hard reset 到目标 commit
5. 更新时间线当前版本状态

要求：

- 回退后，回退前状态必须仍在时间线中
- 用户可以一键回到回退前状态
- hard reset 前必须保证当前未保存状态已保存成功
- 当前无变化时直接回退，不制造空版本
- 不自动推送
- 不修改远端

## 导出

首版用户界面支持「导出压缩包」。

导出规则：

- 从任意一个好版本导出
- 导出对应好版本保存时的项目文件快照
- zip 内包一层项目目录
- 包含 `.env`
- 包含隐藏文件和普通项目文件
- 不包含 `.git/` 目录或 `.git` 指针文件
- 不包含版本历史
- 同名 zip 已存在时自动生成不重名文件
- 导出成功后打开导出目录

后端保留导出当前项目副本能力，当前界面不作为正式用户入口暴露。

## 存储占用

首版展示项目占用大小和版本数据占用大小，包括好版本创建的外置 Git 数据目录占用；不提供清理旧版本能力。

## 本地数据目录

首版提供打开本地数据目录的入口，用于让用户确认版本数据保存在本机。

## MVP 不做事项

- 云同步
- 团队共享
- 自动运行项目
- 自动测试
- Agent 自动联动
- 代码 diff
- 版本搜索
- 删除项目
- 删除旧版本
- 清理存储
- 推送远端
- 分支管理
- 合并冲突处理
- Git 术语教学
- GitHub 登录
- PR / Review 流程
- 权限协作
- Git LFS
- 用编辑器打开项目
- 多语言
